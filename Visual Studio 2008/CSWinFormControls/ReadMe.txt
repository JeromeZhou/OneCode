/////////////////////////////////////////////////////////////////////////////
摘要:

这个示例阐述了怎样自定义Windows Forms控件。


本示例中，有4个小例子：
 
1. 拥有多列的组合框。
   展示了怎样在组合框的下拉列表中显示多列数据。

2. 每个列表项拥有不同提示的列表框。
   展示了怎样为列表框中的每个列表项显示不同的提示。     

3. 只能输入数字的文本框。
   展示了怎样使文本框只允许输入数字。

4. 一个椭圆形的按钮。
   展示了怎样创建一个不规则形状的按钮。 
   
/////////////////////////////////////////////////////////////////////////////
代码逻辑：

1. 例1：拥有多列的组合框
    
    1) 创建一个拥有多列多行的数据表（DataTable）。
    2) 将组合框的数据源绑定到该数据表。
    3) 将组合框的 DrawMode 属性设置为 DrawMode.OwnerDrawFixed;
    4) 处理组合框的 DrawItem 事件。
    5) 在 DrawItem事件中，计算每列的边界同时在边界内绘制相关的值。
    
   
2. 例2：每个列表项拥有不同提示的列表框。

   1) 为列表框添加一些列表项。
   2) 处理列表框的MouseMove事件。
   3) 调用ListBox.IndexFromPoint方法获得鼠标悬停所在列表项的索引。
   4) 如果鼠标悬停在列表项上，调用 ToolTip.SetToolTip 方法显示一个提示。
     
   
3. 例3：只能输入数字的文本框。

   为了使文本框只接收数字，我们需要处理TextBox.KeyPress事件。在该事件处理
   过程中，使用char.IsNumber方法过滤输入的字符。
   
   
4. 例4：一个椭圆形按钮。
   
   创建椭圆形按钮的关键是改变它的Region属性。
