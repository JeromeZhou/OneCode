========================================================================
    控制台应用程序 : VBCheckProcessBitness 概述
========================================================================

/////////////////////////////////////////////////////////////////////////////
摘要:

这个实例代码演示了如何确定一个给定的进程是64位的还是32位的

/////////////////////////////////////////////////////////////////////////////
演示:

通过下面的步骤演示这个例子。

步骤1. 在你在Visual Studio 2008（目标平台是AnyCPU）成功编译这个实例代码后,会得到一个
VBCheckProcessBitness.exe的应用程序。 

步骤2. 在64位操作系统（例如 windows 7 x64）命令提示符（cmd.exe）中运行一个程序。这个程序
会在命令提示符输出如下信息：

  当前进程是一个64位进程。

它指出当前进程是一个64位进程
  
步骤3. 如果你把指定的一个32位进程作为参数传递给VBCheckProcessBitness.exe，程序会在命令提示符输出
如下信息： 
  
  这个进程是32位进程。

它指出给出的进程是一个32位进程。

/////////////////////////////////////////////////////////////////////////////
代码逻辑:

这个实例使用以下逻辑来确定一个进程是否是32位的：
1. 所有运行在32位操作系统下的进程都是32位的。
2. 在64位操作系统下，32位进程只能运行在Wow64里。
  1) 如果这个进程运行在WOw64下，它是一个32位的。
  2) 如果这个进程没有运行在WOw64下，它是一个64位的。
