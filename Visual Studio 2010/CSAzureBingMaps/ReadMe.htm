概述

这是AzureBingMaps的源代码. 你可以在 http://blogs.msdn.com/b/windows-azure-support/archive/2010/08/11/bring-the-clouds-together-azure-bing-maps.aspx 找到相关描述博文. 我们建议您首先阅读博文,作为参考资料来使用这份代码.

项目

解决方案中有6个项目:

  • AzureBingMaps: Windows Azure云项目.
  • AzureBingMaps.WebRole: Web Role项目. 它包含下列组件:
    • WCF数据服务: 在DataService目录下.
    • HTML客户端: HTML界面定义在HtmlClient.aspx中, 相应的JavaScript代码定义在Scripts\HtmlClient.js中.
    • Federation相关代码: 包括LoginPage, FederationCallbackHandler, WindowsLiveProfile, 以及HTML/Silverlight客户端的后台代码.
  • AzureBingMaps.DAL: 数据访问项目,包括SQL脚本和Entity Framework代码.
  • AzureBingMaps.UnitTest: 单元测试项目.
  • SilverlightClient: Silverlight项目.
  • WindowsPhoneClient: Windows Phone客户端.

配置

编译和运行此示例时, 你必须修改配置以提供你自己的Azure, Bing Maps, 以及Windows Live Messenger Connect信息. 一共有7处需要修改的地方:

  • AzureBingMaps项目的ServiceDefinition.cscfg, 配置Windows Azure存储账户,远程桌面身份验证, SSL验证, 以及远程桌面验证.
  • AzureBingMaps项目的Web.config.WebRole, 配置SQL Azure连接字符串, Windows Live Messenger Connect 信息, Windows Azure服务域, AppFabric命名空间, AppFabric 缓存身份验证, 本机密钥, WIF 验证, AppFabric ACS 验证.
  • AzureBingMaps项目的Scripts\HtmlClient.js.WebRole, 配置Bing Maps 身份验证.
  • 如果你想运行单元测试, 配置AzureBingMaps.UnitTest的App.config中的连接字符串.
  • SilverlightClient项目的MainPage.xaml.cs, 配置Bing Maps 身份验证.
  • WindowsPhoneClient项目的MainPage.xaml.cs, 配置Bing Maps 身份验证.
  • WindowsPhoneClient项目的DataSource.cs, 配置WCF 数据服务地址.

此外, 特定的SDK(譬如Windows Live和AppFabric Labs) 并不会将所需的程序集安装到GAC中. 因此当你下载SDK后, 你可能需要修改项目引用以指向正确的程序集引用.
